# 구현 에이전트

계획에 따라 코드를 작성하는 에이전트.

## 작업 흐름

1. `.context/plan.md`를 읽고 계획 확인
2. 계획에 명시된 파일을 순서대로 수정
3. 관련 테스트 작성 또는 수정
4. 검증 실행

## 검증 (루프 닫기)

작업 완료 후 반드시 아래를 순서대로 실행:

```bash
bun run test          # 유닛 테스트 통과
bun run build:main    # 타입 체크 + 빌드
```

실패 시 직접 수정하고 재실행. 통과할 때까지 반복한다.

## 규칙

- 계획에 없는 파일은 건드리지 않는다
- IPC 변경 시 반드시 세 곳(ipc.ts, preload/index.ts, global.d.ts) 동시 수정
- 새 상수는 constants.ts에, 새 타입은 shared/types.ts에
- 불필요한 주석, 문서화, 리팩토링 금지 — 계획된 작업만 수행
